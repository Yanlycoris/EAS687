Introduction:

Blender is a free and open-source 3D computer graphics software toolset used for creating animated films, visual effects, art, 3D printed models, motion graphics, interactive 3D applications, virtual reality, and computer games. (Wikipedia)

BlenderGIS is a free add-on in Blender that used to generate topographies and street maps using data from OpenStreetMap.org and Shuttle Radar Topography Mission (SRTM) data from NASA/Jet Propulsion Laboratory.

Capturing open-source urban spatial data in layers by Blender GIS

In the early stages of planning and design projects, we usually need large-scale urban background information and spatial data. Today this tutorial aims to demonstrate a simple and quick way to obtain open-source urban spatial data, which is already categorized by features and really-to-use for quick site analysis and early design ideation. This method will be of great help to save your time and energy instead of manually tracing the map or digitalizing it in GIS. Enjoy and have fun :)

There are a few checkpoints in this tutorial, reminder to save your file at each checkpoint!

  • Download & Setup (skip this session if you already did! Click the down triangle to hide)

    Download Blender here: https://www.blender.org/download/

    Choose the versions that fit your device. Click the blue-button to download Windows 64 Installer(160MB). If your computer system is macOS or other versions, please click the gray button, then download the correlate Installer in the drop-down menu.

    Download Blender GIS here:

    https://github.com/domlysz/BlenderGIS

    First, click the upright green button "Code", and then click the "Download ZIP" option at the drop-down menu, to download the master folder to your device(about 410KB).

    Find the Blender Installer at your local computer. Double click to launch the Setup Installation.

    Go Next.

    Choose the Location you want to install the Blender software. Then click next.

    Click Install and wait several seconds for the Installation finish.

    Open Blender. Now we will install the Blender GIS Add-on that we download from GitHub before.

    Click edit and then chose Preference at the drop-down menu.

    A new window will pop-up like below.

    Click "Add-on" in left list, then click the install button.

    In the new pop-up window, select the location where you store the Blender GIS folder. (Not need to unzip the file)

    Second, there's another setting that you have to apply: the cache location. Expand the triangle here.

    Scroll down the drop-down menu to the cache folder section and choose a temporary file location to put the cache. That's all the add-on setup! Good job everyone!

Checkpoint #1 Finish Install and Set-up

Back to the Blender main interface. Click the new option that you create that named "GIS"

Navigate to GIS>Basemap.

Click ok to launch the map. It's set to Google but you can also choose OSM/Bing or osm WMS etc. You can also choose the map or the satellite version here. I'm just gonna leave it at the defaults in this demo.

Press OK to start. After the world map loads up, you can zoom in to this map and then find the area that you're interested in.

To demonstrate, I'm gonna zoom in the Detroit City. You can search for a location by press G(keyboard) and type in “Detroit(replace with your destination)” and set a zoom level as well. OR You can also just press the plus icons on the number pad and scrolling to the location.

Once you're happy with it and you can press E (keyboard) to export that section. As an outcome, it brings in a two-dimensional image plane of the area that you selected.

Therefore, you get a high-resolution map for your 3D model.

Navigate to GIS>Web geodata>Get elevation(SRTM)

Press OK to add elevation

Checkpoint #2 Load Basemap + Elevation

Now lets explore the OSM data! Save your file and make sure the basemap is selected. Then navigate to GIS>Web geodata>Get OSM.

Choose the extent first, then select the Ways（data category）you wanted. Also, enable the elevation from objects for obtaining the correct elevation(for example, the height of the building).

**If you think your computer is not strong enough, please try 1-2 Ways each time to avoid a crash.**

You can choose all of them at one time: Holding down shift then select all objects. Then click OK on the bottom. (You might need to be a little bit patient with this. It can take quite a long time, mine took about 10-15 minutes to load up.)

This tutorial will demonstrate the walkthrough process of loading objects separately as follow：

Let‘s add the building layer first. Make sure the basemap is selected, then navigate to GIS>Web geodata>Get OSM, choose building, and make sure enable the checkbox of elevation from objects.

It may take 10mins or more to load up the road/building data. Result of adding building objects:

Only select the basemap, then repeat the steps that navigate to GIS>Web geodata>Get OSM and choose highway this time.

The road data may extend your map extent, but remember to select the same base map extent for other objects.

Repeat the previous steps again for other data categories.

As the result, you can see that we've got lots of different layers here. we've got buildings, highways, land, railway, waterway, etc.

Checkpoint #3 Your Excellent 3D City Model!

After loading all OSM data into your model, you can check those data in the layers panel(in the upright corner), which are categorized in different layers and ready to use.
